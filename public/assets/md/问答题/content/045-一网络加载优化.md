### 一、网络加载优化

#### 1. 减少请求数量

- 合并 CSS/JS 文件
- 使用雪碧图（CSS Sprites）或图标字体（Icon Font）或 SVG Symbol
- 使用 Base64 内联小图片（需权衡缓存与体积）
- 按需加载（懒加载、路由懒加载）

#### 2. 减小资源体积

- 压缩 HTML/CSS/JS（Gzip/Brotli）
- 图片压缩（WebP、AVIF、TinyPNG 等工具）
- 使用 Tree Shaking 删除无用代码（ES Module + Webpack/Rollup）
- 代码分割（Code Splitting）按需加载模块
- 移除 Source Map（生产环境）

#### 3. 利用缓存机制

- 强缓存：Cache-Control / Expires
- 协商缓存：ETag / Last-Modified
- Service Worker 缓存（PWA）
- localStorage / sessionStorage 缓存静态数据
- CDN 缓存静态资源

#### 4. 优化网络传输

- 启用 HTTP/2 或 HTTP/3（多路复用、头部压缩）
- 预加载关键资源：`<link rel="preload">`
- 预连接：`<link rel="preconnect" href="https://example.com">`
- DNS 预解析：`<link rel="dns-prefetch" href="//example.com">`
- 使用 CDN 分发静态资源

#### 5. 资源加载策略

- 关键资源优先加载（Critical CSS / JS）
- 非关键资源延迟加载（async / defer / IntersectionObserver）
- 图片懒加载（`loading="lazy"`）
- 字体加载优化：`font-display: swap`
