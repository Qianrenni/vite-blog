### 🌐 三、后端看到的 `Origin` 是什么？

- **取决于浏览器是否发送了 `Origin` 头**：
  - 同源请求（如 `/api/xxx`）→ 浏览器**不发** `Origin` → 后端看不到。
  - 跨域请求（如 `fetch('http://other:8080')`）→ 浏览器**自动加** `Origin` → Nginx 默认透传 → 后端能看到。
- Nginx “另起 TCP 连接”**不影响 HTTP 头的传递**，它只是传输通道。

> ✅ **最佳实践**：  
> 让前端和 API 共享同一域名（通过 Nginx 代理），后端就**无需处理 CORS**，也看不到 `Origin`。
