### 二、HTTPS 如何保证安全？

HTTPS = HTTP + SSL/TLS，其安全性主要通过以下机制实现：

#### 1. **加密传输（Confidentiality）**

- 使用 **对称加密**（如 AES）加密实际传输的数据。
- 对称密钥通过 **非对称加密**（如 RSA、ECDH）在握手阶段安全协商。
- 即使数据被截获，攻击者也无法解密内容。

#### 2. **身份认证（Authentication）**

- 服务器提供由 **可信 CA（证书颁发机构）** 签发的数字证书。
- 浏览器验证证书的有效性（是否过期、是否被吊销、域名是否匹配、是否由可信 CA 签发）。
- 防止用户连接到假冒网站（如钓鱼网站）。

#### 3. **数据完整性（Integrity）**

- 使用 **消息认证码（MAC）** 或 **AEAD 加密模式**（如 AES-GCM）。
- 确保数据在传输过程中未被篡改。

#### 4. **TLS 握手过程（简化版）**

1. 客户端发起连接，发送支持的加密套件。
2. 服务器返回其 SSL 证书和公钥。
3. 客户端验证证书合法性。
4. 双方协商出一个 **会话密钥（对称密钥）**。
5. 后续通信使用该密钥加密，高效且安全。
