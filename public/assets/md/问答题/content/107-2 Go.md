### 2. Go

#### 原因分析：

- 频繁分配小对象（尤其在高并发 goroutine 中）
- 切片/字符串频繁拷贝
- 逃逸分析失败导致堆分配

#### 应对策略：

- **减少堆分配**：
    - 使用 sync.Pool 复用对象
    - 预分配 slice 容量（make([]T, 0, cap)）
    - 避免在热路径中创建闭包或接口

- **避免内存逃逸**：
    - 尽量让变量在栈上分配（通过逃逸分析：go build -gcflags="-m"）

- **监控 GC 行为**：
  ```go
  import "runtime/debug"
  debug.SetGCPercent(100) // 默认 100%，可调高减少 GC 频率（但增加内存）
  ```

- **使用 pprof 分析内存**：
  ```go
  import _ "net/http/pprof"
  // 访问 /debug/pprof/heap 查看内存分配
  ```
