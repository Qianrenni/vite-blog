### ✅ 二、关键设计要素

#### 1. **选择合适的哈希函数形式**
- **简单累加**（不推荐）：`sum(ord(c)) % size`  
  → 忽略字符顺序，冲突高。
- **多项式滚动哈希**（推荐）：  
  ```python
  h = 0
  for c in s:
      h = (h * base + ord(c)) % size
  ```
  → 考虑字符位置和值，分布更均匀。

#### 2. **合理选择基数（base）**
- 推荐使用**质数**，如 31、37、131。
- **必须与数组大小互质**（gcd(base, size) = 1），避免信息丢失。
- 避免使用 2、10 等常见合数（尤其当 size 是 2 或 5 的倍数时）。

#### 3. **数组大小（size）的选择**
- **理想情况**：选**质数**（如 101 而非 100），可大幅降低周期性冲突。
- 若 size 固定为合数（如 100），需更谨慎选择 base（避开其质因数：2 和 5）。

#### 4. **取模时机**
- 可在每一步取模（防溢出，适合长字符串）；
- 也可最后统一取模（短字符串可接受）。
